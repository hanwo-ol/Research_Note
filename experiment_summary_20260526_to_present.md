# U-Net Skip Connection 구조적 결함 추적 및 TESC 설계 요약 (2026-05-26 ~ 현재)

## 0. 실험 배경 및 목표
* **배경**: ST-DiT의 Raw Pixel 학습에서, 디코더의 마지막 계층에서 고주파 성분(선명도)이 뭉개지는 현상이 관찰되었습니다. 이를 타개하기 위해 U-Net 구조처럼 인코더(Stem)의 초기 고주파 특징을 디코더로 직접 넘겨주는 **Deep Connection (U-Net Skip Connection)** 도입을 시도했습니다.
* **핵심 목표**: 트레이드 오프(MSE 손상) 없이 고해상도 픽셀 정보만을 안정적으로 합성하여, 최종 이미지의 화질(FID, LPIPS)을 베이스라인(FID ~30대) 이상으로 극대화하는 것.

## 2. 단계별 실험 목표, 결과 및 인사이트 (엄밀한 분석)

### 2.1. Sigmoid / Softplus 게이팅 실험
* **목표**: 스킵 커넥션이 초기 학습에 미치는 충격(Loss 폭발)을 막기 위해, 비선형 함수(Sigmoid, Softplus)를 사용하여 게이트를 닫힌 상태(`~0.01`)로 초기화(`bias=-4.5`)한 뒤 서서히 열리도록 유도.
* **결과**: 두 실험 모두 게이트 값이 `0.012` 등에서 더 이상 상승하지 않고 정체되는 현상 발생. 성능 역시 베이스라인에서 전혀 개선되지 않음.
* **인사이트 (Dead Gradient)**: `bias=-4.5`라는 극단적인 초기값 때문에 비선형 함수의 미분값(Gradient)이 사실상 `0`에 수렴했습니다. (예: `dL/dy * 0.01`). 즉, **네트워크가 스킵 커넥션의 필요성을 깨닫더라도 게이트를 열 물리적 힘(Gradient)이 전달되지 못하는 설계적 결함**이었습니다.

### 2.2. Zero-Init (Direct LayerScale) 실험
* **목표**: Dead Gradient를 해결하기 위해 비선형 함수를 아예 제거하고, 파라미터를 그대로 곱해주는 Direct LayerScale(`x = x + gate * skip`)을 도입. 초기값을 완벽한 `0.0`으로 설정.
* **결과**: 초기 학습 충격 없이 안정적으로 출발했으며, 게이트가 `[0.302, 0.342, 0.589]`까지 시원하게 열렸습니다. **그러나, 최종 FID가 110.56으로 끔찍하게 무너졌습니다.**
* **인사이트 (Noise Injection Issue)**: 게이트가 열렸음에도 화질이 파탄 난 것은 **'타임스텝(t) 정보의 부재'** 때문이었습니다.
  디퓨전 모델은 $t=1000$일 때 순수 노이즈를, $t=0$일 때 깨끗한 이미지를 다룹니다. 하지만 디코더의 스킵 커넥션은 $t$가 무엇이든 항상 인코더의 특징(노이즈 포함)을 30~50%씩 결과물($x_0$ 예측)에 더해버렸습니다. 즉, **노이즈를 예측해야 할 시점에 노이즈 자체를 정답에 섞어버리는 구조적 폭력**이 가해진 것입니다.

### 2.3. TASC (Timestep-Adaptive Skip Connection) 실험
* **목표**: 타임스텝($t$)에 따라 게이트 개방도를 능동적으로 조절하기 위해, $t$의 임베딩(`t_emb`)을 입력으로 받아 게이트 값을 출력하는 초경량 `MLP`를 도입. ($t$가 클 때는 닫고, 작을 때는 열기를 기대함)
* **결과**: 모델 학습은 완료(Val MSE 2.87)되었고 FID는 99.48로 약간 개선되었으나 여전히 베이스라인(30대)에 크게 미달. 게이트 가중치 추출 결과, 각 레이어의 게이트가 `[-34.5, 3.8, 20.2]`라는 극단적이고 폭발적인 수치로 굳어져 있었습니다.
* **인사이트 (Encoder Collapse)**: 
  `t_emb`는 정규화되지 않은 512차원 벡터입니다. 여기에 연결된 MLP는 학습 첫 스텝부터 엄청난 크기의 Gradient Explosion을 겪으며 게이트를 34배 크기로 열어버렸습니다.
  게이트가 34배 열리면 $t=1000$의 노이즈가 34배 증폭되어 쏟아집니다. Loss 폭발을 직감한 네트워크는 게이트를 다시 닫는 정상적인 방법 대신, **입력을 제공하는 인코더(Stem)의 출력을 스스로 0으로 죽여버리는(Collapse) 극단적인 지름길**을 택했습니다. 인코더가 죽어버렸으니 디코더 메인 패스로 가야 할 공간 정보까지 소멸되어 FID가 망가진 것입니다. (빈대 잡으려다 초가삼간을 태운 전형적인 사례)

### 2.4. TESC (Timestep-Enveloped Skip Connection) 실험 (현재 진행 중)
* **목표**: 학습 가능한 MLP에 의존하지 않고, 확정적인 수학적 봉투 함수(Envelope)를 사용하여 노이즈 유입과 인코더 붕괴를 원천적으로 차단.
* **구현 방식**: 게이트는 가장 안정적인 상수 파라미터(`0.0` 초기화)로 되돌리되, 텐서 합산 시 `(1.0 - t/1000)` 수식을 강제로 곱해줍니다.
  * **$t=1000$**: Envelope이 `0.0`이 되어 게이트가 물리적으로 완벽 차단됨 $\to$ 인코더가 자신을 붕괴시킬 이유 상실.
  * **$t=0$**: Envelope이 `1.0`이 되어 파라미터가 학습한 스킵 커넥션 비율 100% 개방.
* **기대 효과**: 연산량 증가나 파라미터 폭발 위험 없이, 안전하게 고주파 특징만을 $t=0$ 부근에서 디코더에 제공할 수 있습니다.

## 4. 중대 발견: Noise Feature Injection 구조적 오류와 해결책 (CCDC)
TESC(타임스텝에 따른 완벽한 게이트 차단)를 적용했음에도 FID가 104.16으로 붕괴하는 현상을 추적한 결과, 아키텍처의 근본적인 **배관(Plumbing) 오류**를 발견했습니다.

* **원인 (Noise Feature Injection)**: 현재 디코더로 전달되는 `skip_features`는 `x_target`을 처리하는 `stem`에서 추출됩니다. 여기서 `x_target`은 디퓨전의 **노이즈가 낀 이미지($x_t$)**입니다. 즉, 디코더는 깨끗한 최종 예측($x_0$)을 만들어내야 하는데, 바로 직전 계층에서 **노이즈가 가득한 고주파 특징을 무차별적으로 쑤셔 넣고 있었던 것**입니다! $t=1000$에서 게이트를 차단하더라도, $t=900, 800$ 등 여전히 노이즈가 강한 구간에서 노이즈 특징이 주입되어 최종 화질을 파괴했습니다. (DDPM처럼 노이즈($\epsilon$)를 예측하는 모델에서는 이 방식이 맞지만, $x_0$ 예측 모델에서는 치명적입니다.)

* **해결책 (Clean Context Deep Connection; CCDC)**: 
  다행히 우리 모델은 과거 관측 프레임인 `x_context`를 처리하는 쌍둥이 스템 `context_embedder`를 가지고 있습니다. `x_context`는 노이즈가 전혀 없는 **완벽히 깨끗한 과거 이미지**입니다. 
  따라서 `x_target`의 노이즈 특징 대신, **가장 최근의 깨끗한 과거 프레임(`x_context`의 마지막 프레임)에서 추출한 `skip_features`를 디코더로 전달**하면 모든 문제가 해결됩니다! 깨끗한 구름의 에지와 형태가 그대로 디코더로 넘어가므로, 디코더가 $x_0$를 합성할 때 완벽한 고해상도 가이드라인 역할을 할 수 있습니다.

## 5. 총괄 결론 및 향후 방향
5월 26일부터 시작된 일련의 U-Net Skip Connection 도입 시도는, 단순히 텐서를 더하는 행위가 디퓨전 모델의 타임스텝 물리학(Timestep Physics)과 예측 목표($x_0$ vs $\epsilon$)에 어긋날 때 얼마나 치명적인 부작용(Dead Gradient $\to$ Encoder Collapse $\to$ Noise Injection)을 낳는지 증명하는 과정이었습니다.

이제 노이즈를 섞는 치명적 오류를 바로잡고, 깨끗한 과거 프레임의 고주파 정보를 제공하는 **CCDC (Clean Context Deep Connection)** 로 전환할 준비가 되었습니다.
